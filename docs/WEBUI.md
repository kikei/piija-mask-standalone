# WEB UI 仕様

目的: ユーザーが LLM へ貼り付ける前に、個人情報を最小構成で安全にマスクする。

## 1. 画面構成

### 1.1 入力エリア

- 単一のテキストエリアを配置する。
- プレースホルダ例: ここに文章を貼り付け、「マスクする」を押してください。

### 1.2 操作ボタン

- マスクする
- コピー
- 入力に戻る（右上の小さなテキストボタン）

### 1.3 表示エリア

- タブ切替: 元の文章 / マスク後の文章
- マスク後の文章にはマスク適用後のテキストを表示する。
- マスク箇所は全角「＊＊＊＊」で表示し、クリックで解除できる。

### 1.4 常時表示メッセージ

- マウスでドラッグすると追加でマスクできます。マスク部分をクリックすると解除します。

## 2. 操作仕様

### 2.1 基本フロー

1. 入力エリアに文章を貼り付ける。
2. マスクするを押して自動マスクを実行する。
3. 必要に応じて手動で追加マスクや解除を行う。
4. コピーを押して、マスク後の文章の表示どおりにコピーする。

### 2.2 追加マスク（ドラッグ終点で適用）

- マスク後の文章上で文字範囲をドラッグ選択し、選択が終わった時点（マウスアップ）でその範囲を「＊＊＊＊」に置換する。
- 選択中は置換を行わない（選択中に文字位置が動かないようにする）。
- 追加マスク後は、表示中テキスト内の同一文字列（完全一致）をすべて「＊＊＊＊」に置換する。

### 2.3 マスク解除（クリックのみ）

- マスク後の文章上の「＊＊＊＊」をクリックすると、その箇所だけ原文に戻す。
- 再度マスクしたい場合は、その範囲をドラッグして追加マスクを行う。
- 解除履歴や学習は保持しない。

### 2.4 入力に戻る

- 押下時に、マスク後の文章を自動マスク結果にリセットする。
- 手動での追加マスクや解除はすべて破棄する。

### 2.5 コピー

- クリップボードへ、マスク後の文章の表示内容をそのまま書き込む。
- 失敗時の案内文: 自動コピーが無効です。表示中の結果を選択して、手動でコピーしてください。

## 3. 文言仕様

### 3.1 ボタン・タブ

- マスクする
- コピー
- 入力に戻る
- 元の文章 / マスク後の文章

### 3.2 プレースホルダ

- ここに文章を貼り付け、「マスクする」を押してください

### 3.3 エラーメッセージ（行動を促す）

- 入力が空のとき: マスクしたい文章を貼り付けてください。
- クリップボード失敗のとき: 自動コピーが無効です。表示中の結果を選択して、手動でコピーしてください。

## 4. 表示仕様

### 4.1 マスク表現

- 置換文字は全角「＊」とする。
- 検出または選択したひとかたまりは、常に「＊＊＊＊」に置換する（文字数は保持しない）。
- 部分的に残す処理は行わない。

### 4.2 クリック可能領域

- 各マスクブロックはクリックで原文と「＊＊＊＊」をトグル表示する（このセッション内のみ）。
- 原文はメモリ上だけに保持し、送信や保存は行わない。
- 隣接するマスクブロックは連結しない。

### 4.3 追加マスクの一括反映

- 追加マスク実行後は、表示中テキスト内の同一文字列（完全一致）も一括で「＊＊＊＊」に置換する。

## 5. やらないこと

- 本文は保存しない。
- 学習しない。
