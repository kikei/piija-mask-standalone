# アーキテクチャ指針

この文書は、本プロダクトをブラウザ内で個人情報を自動マスクできる最小構成の Web アプリとして設計し、実装するための大まかな方針を示します。TypeScript と ESM を前提に、依存を最小化しながら長期にわたって無理なく保守できる構成を目指します。将来的な拡張として高度解析用のサーバ連携を想定しますが、初期段階ではオフライン完結の体験を最優先します。

## パッケージ構成

### @redactjp/redact

抽出と置換のロジックに加えて人名検出 (kuromoji) を集約したライブラリとして提供します。ブラウザと Node の両方で動作し、型はこのパッケージから直接エクスポートします。

### @redactjp/web-ui

最小の Web クライアントとして動作し、実行時に `@redactjp/redact` を利用します。人名検出用の辞書は画面表示後に自動で遅延ロードします。

## 設計原則

- アプリは静的配信を採用し、単一 HTML、ES Modules、素の CSS で構成します。
- Web アプリケーションフレームワークや PWA は使用しません。
- 言語は TypeScript を採用し、API は ESM を前提とします。
- 既定の運用はサーバ不要のオフライン完結とし、サーバ連携は別設計で段階的に追加します。
- ユニットテストは Node の内蔵テストランナーと tsx を用いて実行し、ESM と TypeScript に素直な最小設定を維持します。
- 仕様の詳細は MASK.md と WEBUI.md を一次情報として管理し、本書は指針レベルの内容に留めます。

## 実行時フロー (概要)

1. ページ表示直後に人名辞書の初期化を非同期で開始します。
2. ユーザーが操作した時点でマスク処理を実行します。
3. 人名辞書の準備が整い次第、人名の追加マスクを順次反映します。
4. 追加マスクや解除などの操作は Web UI 側で完結させ、詳細は WEBUI.md の仕様に従います。

## ビルドと配布の方針

- TypeScript の型チェックは `tsc` で行います。
- JavaScript のバンドルと最小化は esbuild で実施します。
- 配信は静的ホスティングで行い、辞書は静的ファイルとして提供してブラウザキャッシュを前提に運用します。
- 具体的なビルド設定や命名は実装段階で合意し、必要に応じて見直します。

## セキュリティとプライバシー (指針)

- 本文は送信も保存も学習もせず、処理はブラウザ内で完結させます。
- 原文表示のトグルはメモリ内の情報に限定し、永続化を行いません。
- 第三者ライセンスの表記は `THIRD-PARTY-NOTICES.md` に同梱し、配布物に含めます。

## バージョニング

- 初期段階では 0.x 系で素早く変更を取り込み、`@redactjp/redact` の API が安定した段階で 1.0.0 に到達させます。
- `@redactjp/web-ui` は `@redactjp/redact` の安定版に追随し、互換性を維持します。

## 将来拡張

- 必要に応じて Web Worker 化や辞書分割などの最適化を検討します。
- サーバ側 API や SDK の追加は別設計として段階的に導入します。
- 検出対象や対応言語の拡張は仕様の更新とあわせて計画します。
